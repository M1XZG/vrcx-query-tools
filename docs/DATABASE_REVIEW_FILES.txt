================================================================================
YOUR USE CASE: HOUR-BY-HOUR PEOPLE COUNTS
================================================================================

Goal: Create a spreadsheet showing how many people were in your VRChat 
      instances hour-by-hour throughout the day

Solution: Run vrcx_query.py or vrcx_query.js
          Both scripts automatically generate Excel files with:
          - Hour of day
          - Instance ID
          - World name
          - Number of joins
          - Number of leaves
          - Unique people count

Time to Results: 5 minutes

================================================================================
DATABASE SUMMARY
================================================================================

Type: SQLite 3
Location: %APPDATA%\VRCX\VRCX.sqlite3

Key Tables:
  ✓ gamelog_location - Your location timeline
  ✓ gamelog_join_leave - People joining/leaving your instances
  ✓ {userPrefix}_feed_* - Friend activity feeds
  ✓ {userPrefix}_friend_log_* - Friend tracking
  ✓ cache_avatar / cache_world - Metadata caches

Total Tables: 20+
Total Columns: 100+

================================================================================
QUICK START (5 MINUTES)
================================================================================

Step 1: Close VRCX Application
        (Database is locked while VRCX runs)

Step 2: Install Dependencies
        pip install pandas openpyxl

Step 3: Run Script
        python vrcx_query.py

Step 4: Open Excel File
        vrcx_exports/vrcx_hourly_YYYY-MM-DD.xlsx

Done! You now have your hour-by-hour spreadsheet.

================================================================================
TROUBLESHOOTING
================================================================================

Database Locked:
  → Close VRCX application and try again

Database Not Found:
  → Set VRCX_DATABASE_PATH environment variable

Missing Dependencies:
  → pip install pandas openpyxl
  → npm install better-sqlite3

No Data Found:
  → VRCX must have been running on that date
  → Try a recent date
  → Check database integrity
